**WinSCP — SFTP/FTP/SCP-клиент для Windows. Основное назначение — безопасная передача файлов между компьютерами и серверами.**

## **Что куда и зачем:**

### **Основные компоненты:**
- **Левый раздел** — локальный компьютер
- **Правый раздел** — удаленный сервер
- **Панель логирования** — журнал операций
- **Панель очереди** — статус передачи файлов

### **Протоколы:**
1. **SFTP** (рекомендуется) — через SSH-соединение, порт 22
2. **FTP** — стандартный, порт 21
3. **FTPS** — FTP с шифрованием
4. **SCP** — копирование через SSH

### **Практические примеры:**

#### **1. Подключение к Linux-серверу:**
```
Хост: 192.168.1.100 или domain.com
Порт: 22
Имя пользователя: root или user1
Пароль: ТУТ_ПАРОЛЬ
Протокол: SFTP
```

#### **2. Автоматизация через скрипты:**
```batch
winscp.com /command ^
    "open sftp://user:password@server.com/" ^
    "put C:\local\file.txt /remote/path/" ^
    "exit"
```

#### **3. Синхронизация директорий:**
```ini
open sftp://user@server.com
synchronize remote D:\project\ /var/www/
exit
```

### **Ключевые операции:**
- **Загрузка на сервер**: выделить файлы → правый клик → Закачать
- **Скачивание с сервера**: выделить файлы → правый клик → Скачать
- **Прямое редактирование**: двойной клик по файлу на сервере
- **Синхронизация**: Команда → Синхронизация

### **Для разработчиков:**
- Работа с файлами на тестовых серверах
- Развертывание кода (deploy)
- Резервное копирование баз данных
- Мониторинг логов

### **Безопасность:**
- Используйте SSH-ключи вместо паролей
- Проверяйте fingerprint сервера при первом подключении
- Не сохраняйте пароли в сессиях на общих ПК

**WinSCP заменяет команды типа `scp`, `sftp` в Linux, предоставляя графический интерфейс и возможности автоматизации через скрипты.**
